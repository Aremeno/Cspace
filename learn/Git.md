## 前提知识

### 基本概念

#### 三种状态  

- 已提交
  
    已提交表示数据已经安全地保存在本地数据库中。
- 已修改
    已修改表示修改了文件，但还没保存到数据库中。
- 已暂存
    已暂存表示对一个已修改文件的当前版本做了标记，使之包含在下次提交的快照中。

#### 状态判断

- 已提交状态：
  Git仓库中存放着特定版本的文件
- 已暂存状态：
  文件已修改且存入了暂存区
- 已修改状态：
  文件只进行了修改  

#### 三种基本区域

1. 工作目录  
  工作区是对项目的某个版本独立提取出来的内容。 这些从 Git 仓库的压缩数据库中提取出来的文件，放在磁盘上供你使用或修改。
2. 暂存区域
  暂存区是一个文件，保存了下次将要提交的文件列表信息，一般在 Git 仓库目录中。 按照 Git 的术语叫做“索引”，不过一般说法还是叫“暂存区”。
3. Git仓库
  Git 仓库目录就是用来存储数据的地方。 这是 Git 中最重要的部分，从其它计算机克隆仓库时，复制的就是这里的数据

#### Git基本工作流程

1. 在工作区中修改文件。
2. 将你想要下次提交的更改选择性地暂存（意思是仅将更改部分暂存，而仓库中的数据不变）
3. 提交更新，找到暂存区的文件，将快照永久性存储到 Git 目录。(检查后，确定更改，那么就可以把确定的版本存入仓库中)
